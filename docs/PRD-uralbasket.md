# PRD — Сайт мастеров лозоплетения (UralBasket)

## Problem Statement

Два мастера по изготовлению изделий из лозы (корзины, декор, элементы мебели) не имеют собственного сайта. Потенциальные клиенты не могут увидеть ассортимент, узнать о мастерах и оформить заказ онлайн. Мастера теряют клиентов из-за отсутствия цифрового присутствия.

## Solution

Создать многостраничный статический сайт на чистом HTML/CSS/JS, развёрнутый на Netlify. Сайт представляет работы мастеров по категориям, содержит блог со статьями, страницу «О нас» с контактами и форму заказа для отдельных товаров, отправляющую данные на email владельцев через Formspree.

## User Stories

### Посетитель / потенциальный покупатель

1. Как посетитель, я хочу увидеть красивую главную страницу с фотографиями работ, чтобы сразу понять, чем занимаются мастера.
2. Как посетитель, я хочу видеть навигацию с разделами (Главная, Каталог, О нас, Блог), чтобы быстро перейти к нужному разделу.
3. Как посетитель, я хочу просматривать товары по категориям (например, Корзины / Мебель / Декор), чтобы найти нужный тип изделия.
4. Как посетитель, я хочу видеть карточку товара с фотографией, названием и описанием, чтобы оценить изделие.
5. Как посетитель, я хочу кликнуть на фото товара и открыть его в lightbox (увеличенный вид), чтобы рассмотреть детали.
6. Как посетитель, я хочу нажать кнопку «Заказать» на карточке товара и заполнить форму (имя, телефон, email, комментарий), чтобы оставить заявку на изготовление.
7. Как посетитель, я хочу видеть название товара предзаполненным в форме заказа, чтобы не вводить его вручную.
8. Как посетитель, я хочу получить подтверждение (сообщение на странице), что моя заявка отправлена.
9. Как посетитель, я хочу видеть товары без кнопки «Заказать» как примеры работ (без лишних объяснений), чтобы понять разнообразие работ мастеров.
10. Как посетитель, я хочу прочитать статью в блоге о мастерах или их процессе, чтобы лучше познакомиться с ними.
11. Как посетитель, я хочу видеть список статей блога с заголовком, датой и кратким описанием, чтобы выбрать интересующую.
12. Как посетитель, я хочу прочитать страницу «О нас» с историей мастеров, фотографиями и их специализацией.
13. Как посетитель, я хочу найти контактные данные (телефон, email, соцсети) на странице «О нас» и в подвале сайта.
14. Как посетитель с мобильного телефона, я хочу, чтобы сайт корректно отображался на маленьком экране.
15. Как посетитель, я хочу видеть сайт только на русском языке.

### Владелец / мастер

16. Как владелец, я хочу получать email с данными покупателя и названием товара при каждом заказе, чтобы связаться с клиентом.
17. Как владелец, я хочу чтобы форма отправлялась через Formspree без необходимости поддерживать серверную часть.
18. Как владелец, я хочу иметь возможность пометить товар как «недоступный для заказа», просто убрав кнопку «Заказать» в HTML.

## Implementation Decisions

### Архитектура

- **Тип сайта:** Полностью статический, многостраничный (MPA — multiple pages, separate HTML files)
- **Стек:** HTML5, CSS3, Vanilla JavaScript (без фреймворков)
- **Хостинг:** Netlify (бесплатный тариф, деплой через Git)
- **Форма заказа:** Formspree — submit endpoint, без бэкенда

### Структура страниц

- `index.html` — главная страница: hero-секция, краткий preview каталога, блок «О нас», последние статьи блога, контакты в футере
- `catalog.html` — каталог товаров с фильтрацией по категориям (2–4 категории)
- `about.html` — страница «О нас» с историей мастеров, фото, контактами
- `blog/index.html` — список статей
- `blog/article-slug.html` — отдельные страницы статей (по одному файлу на статью)

### Модули

1. **Навигация** — общий header с логотипом и ссылками, подключается через JS fetch/include или дублируется в каждом HTML
2. **Каталог товаров** — сетка карточек (CSS Grid), фильтр по категориям (JS, без перезагрузки страницы)
3. **Карточка товара** — изображение, название, описание, опциональная кнопка «Заказать»
4. **Lightbox** — нативный JS lightbox без внешних библиотек (или легковесная библиотека типа GLightbox)
5. **Модальная форма заказа** — overlay с формой, поле «Товар» заполняется названием автоматически, submit через Formspree
6. **Блог** — статические HTML-страницы, список статей с карточками
7. **Футер** — контакты, ссылки на соцсети, копирайт

### CSS-дизайн

- Цветовая палитра: тёплые земляные тона (бежевый, коричневый, терракота, молочный белый)
- Типографика: шрифт с засечками для заголовков (Google Fonts: Playfair Display), sans-serif для текста
- Адаптивный дизайн: mobile-first, breakpoints для планшета и десктопа

### Форма заказа

- При нажатии «Заказать» открывается модальное окно
- Поля: Имя (обязательное), Телефон (обязательное), Email, Комментарий, скрытое поле «Товар» (заполняется JS)
- Submit URL: Formspree endpoint (настраивается в HTML)
- После успешной отправки — скрыть форму, показать сообщение «Спасибо, мы свяжемся с вами!»

### Контакты

- Отображаются на странице «О нас» (секция в конце)
- Продублированы в футере на всех страницах
- Включают: телефон, email, ссылки на VK / Instagram / Telegram (по наличию)

## Testing Decisions

- Хороший тест проверяет внешнее поведение, а не детали реализации
- **Форма заказа:** при клике «Заказать» открывается модал, поле «Товар» заполнено, после submit отображается сообщение успеха
- **Фильтрация каталога:** при выборе категории показываются только карточки этой категории
- **Lightbox:** при клике на изображение открывается lightbox с верным фото
- **Ручное тестирование:** просмотр на мобильном, планшете и десктопе; проверка отправки формы через реальный Formspree endpoint
- Автоматизированные E2E тесты (Playwright) — опционально, не обязательно для MVP

## Out of Scope

- Панель администратора / CMS для редактирования контента без кода
- Корзина покупок и онлайн-оплата
- Поиск по сайту
- Многоязычность
- Личный кабинет покупателя
- Расширенная SEO-оптимизация (только базовые meta-теги)
- Автоматический деплой без участия разработчика

## Further Notes

- Название проекта: **UralBasket** (рабочее)
- Мастера работают вместе как один бренд — отдельных страниц мастеров нет, только общий «О нас»
- Контент (фото, тексты товаров, статьи) предоставляется заказчиком
- Добавление новых товаров и статей выполняется разработчиком через правку HTML-файлов
